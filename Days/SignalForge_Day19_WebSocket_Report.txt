2. Current behavior / architecture details

Architecture:
- Browser (React) → WebSocket → FastAPI backend → ConnectionManager broadcasts to local clients
- When multiple backend workers exist, Redis pub/sub bridges between them
- Each worker has its own ConnectionManager with local WebSocket connections
- Redis pub/sub ensures all workers receive the event and can broadcast to their clients

Fallback behavior:
- If WebSocket connection fails (e.g. server not running), the frontend silently ignores the error
- TanStack Query's `refetchInterval: 3000` continues polling every 3 seconds as a fallback
- When WebSocket IS connected, incidents appear on the dashboard within milliseconds of creation/update

Test safety:
- The `asyncio.create_task(broadcast_incident_event(...))` call is wrapped in try/except
- In tests (no running event loop), the broadcast is skipped and the incident is still created/updated normally
- This means all 14 existing tests pass without modification

5. Beginner-Friendly Explanation

Before Day 19, the dashboard checked for new incidents every 3 seconds by asking the server "are there any new incidents?" This is called polling. It works but it's slow — you might wait up to 3 seconds before seeing a new incident.

WebSockets are different. Instead of the browser asking the server repeatedly, the browser opens a persistent connection and says "tell me when something happens." When the server creates a new incident, it immediately sends a message through the WebSocket to the browser. The browser then refreshes the incident list.

This is like the difference between checking your email every 3 minutes vs having push notifications on your phone. WebSockets are push notifications for the web.

The system also uses Redis pub/sub as a message bus. If you have multiple backend servers (e.g. for load balancing), Redis makes sure all of them know about the incident so they can all tell their connected browsers.

If the WebSocket connection fails (e.g. the server is down), the dashboard falls back to polling every 3 seconds. So even if WebSockets don't work, the dashboard still works.
