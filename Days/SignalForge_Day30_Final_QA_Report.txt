Resume bullets (final)
----------------------

- Built SignalForge, a production-ready incident management platform for
  microservices, in 30 days using FastAPI, React, PostgreSQL, Redis, and Kafka
- Architected a decoupled ingestion pipeline: API returns 202 Accepted in ~15ms,
  worker processors handle DB writes, anomaly detection, and incident creation
  asynchronously via Kafka consumer groups
- Implemented rolling-window anomaly detection with explainable thresholds
  (error rate, p95 latency) and structured incident evidence with timeline
- Designed a 5-dimension rule-based root-cause engine (deployment recency,
  anomaly severity, log errors, trace failures, runbook coverage) with
  traceable evidence per dimension
- Added AI triage with LLM augmentation (OpenAI → mock fallback) producing
  structured output: summary, likely causes, evidence points, suggested actions,
  confidence score
- Built semantic search via pgvector embeddings for concept search across
  incidents and operational runbooks
- Implemented API key authentication with tenant isolation enforced at the
  storage layer — every query filters by authenticated tenant_id
- Added production hardening: structured logging with request tracing,
  in-memory sliding-window rate limiting (100 RPS), safe error handling
  (no stack traces in production), enhanced health checks with dependency
  connectivity tests
- Created a deterministic demo seed script that generates a complete incident
  story (50 events, 1 critical incident, 2 runbooks, deployment correlation)
  in 2 seconds for repeatable interview demos
- Wrote comprehensive documentation: README with architecture diagram and
  scalability discussion, DEMO.md with 8-step walkthrough, AWS_ARCHITECTURE.md
  with full deployment spec, 30 daily reports
- Wrote 57 tests covering anomaly detection, event processing, incident
  lifecycle, integration ingest-to-incident, service graph, runbook CRUD,
  search, auth, and health checks — all passing in <2 seconds
- Load tested at 47.7 RPS on SQLite, identified SQLite single-writer lock as
  bottleneck, documented upgrade path to PostgreSQL for 500+ RPS
- Containerized full stack with Docker Compose: backend (FastAPI), frontend
