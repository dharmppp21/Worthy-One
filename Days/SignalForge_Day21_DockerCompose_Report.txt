2. Current behavior / architecture details

Docker Compose startup:
```powershell
cd .\signalforge_mvp
docker-compose up -d
```

This single command starts all 4 services in the correct order:
1. PostgreSQL starts first, health check passes when `pg_isready` succeeds
2. Redis starts second, health check passes when `redis-cli ping` returns PONG
3. Redpanda starts third, health check passes when `rpk cluster info` succeeds
4. Backend starts last, only after all three dependencies are healthy

The backend container automatically:
- Runs Alembic migrations on startup (`alembic upgrade head`)
- Starts the Kafka consumer worker in a background thread
- Registers all 11 routers
- Serves the API on port 8000

Access everything:
- API docs: http://localhost:8000/docs
- Health check: http://localhost:8000/health
- Frontend: http://localhost:5173 (run separately with `npm run dev`)

5. Beginner-Friendly Explanation

Docker Compose is a tool that lets you define and run multiple containers with a single command. Instead of starting PostgreSQL, Redis, Kafka, and the backend separately, you write a `docker-compose.yml` file that says "start all of these together, and make sure they can talk to each other."

The `docker-compose.yml` for SignalForge defines 4 services:
1. PostgreSQL — the database that stores all events, incidents, and runbooks permanently
2. Redis — a fast in-memory store that keeps the last 50 events per service for quick anomaly detection
3. Redpanda — a lightweight Kafka alternative that handles event streaming
4. Backend — the FastAPI app that receives events, processes them, and serves the API

Each service has a health check. The backend won't start until PostgreSQL, Redis, and Redpanda are all ready. This prevents race conditions where the backend tries to connect to a database that isn't up yet.

All services use "named volumes" — this means even if you stop the containers, your data survives. When you start them again, PostgreSQL still has all the events and incidents from before.

The README is the documentation that tells anyone how to use the project. It includes:
- How to start everything with one command
- How to test the API
- What each endpoint does
- How the system works internally
- What to do next if you want to extend it
