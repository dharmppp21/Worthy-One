2. Current behavior / architecture details

Provider chain:
1. OpenAIProvider: If `OPENAI_API_KEY` is set, uses `gpt-4o-mini` with `response_format={"type": "json_object"}` and temperature=0.2 for deterministic outputs. The prompt includes only the incident's timeline evidence (anomaly stats, deployments, rolling window context). The model is explicitly instructed NOT to hallucinate causes or metrics not in the evidence.
2. MockProvider: If OpenAI is unavailable, this deterministic provider generates structured responses based on patterns in the incident data. For example, if a deployment is in the timeline and error rates are high, it suggests "Recent deployment introduced a regression." It always produces valid AITriageResponse structures.
3. Unavailable fallback: If both providers fail, returns a structured response with a clear message and a reference to the rule-based root-cause endpoint.

Design principles:
- The LLM is NOT the source of truth. The response includes a mandatory disclaimer.
- The AI only analyzes provided evidence. The prompt explicitly forbids hallucination.
- The mock provider is deterministic and reliable for tests and demos. It doesn't require API keys or network calls.
- The AI triage is layered ON TOP of the rule-based root-cause engine (Day 17), not replacing it. Both panels appear in the incident detail.

5. Beginner-Friendly Explanation

The rule-based root-cause engine (Day 17) gives you a structured score with clear evidence. But it's just numbers and short strings. The AI triage layer adds natural-language understanding.

When you open an incident, the AI reads the same evidence the rule engine uses (anomaly stats, deployments, rolling window) but describes it in plain English. It might say: "Anomaly in checkout-service correlated with recent deployment v42. Elevated error rate suggests a code regression. High latency indicates resource contention."

The AI has two modes:
1. OpenAI mode: If you set an API key, it uses GPT-4o-mini to generate the analysis. It's smart but requires a key and internet.
2. Mock mode: If no key is set, it uses a built-in pattern matcher. It's not as creative, but it's free, fast, and always works.

Most importantly, the AI is NOT the boss. Every response includes a disclaimer telling you to verify the analysis against real system evidence. The rule-based engine (Day 17) is still there as the ground truth. The AI is just a translator.
