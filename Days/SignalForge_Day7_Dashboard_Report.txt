Polish the React dashboard for incidents: incident cards, severity colors, incident details, timeline view, empty/loading/error states, and clean responsive layout.


1. What Changed

The frontend was upgraded from a basic incident list to a full-featured, demo-ready incident triage dashboard.

Updated files:

- signalforge_mvp/frontend/src/types.ts
- signalforge_mvp/frontend/src/api.ts

2. Dashboard Components Added

Incident Cards (IncidentCard component):
- Color-coded left border: red for critical, orange for warning, blue for info.
- Severity badge with icon (🔴 critical, 🟡 warning, 🔵 info).
- Status badge: investigating, mitigated, resolved.
- Relative time display (e.g., "2m ago", "just now").
- Service name tag and evidence count.
- Click to open detail panel.
- Hover lift animation for interactivity.

Incident Detail Panel (IncidentDetail component):
- Slide-in overlay from the right (mobile-friendly).
- Full incident summary, service name, and all evidence items.
- Interactive timeline showing every event in the incident lifecycle.
- Status action buttons: mark as investigating, mitigated, or resolved.
- Live status update via PATCH API with auto-refresh after change.

Timeline (TimelineEntry component):
- Vertical timeline with connecting line.
- Icon per event type: 🚨 created, 📝 status_changed, 📊 evidence_added.
- Timestamp, message, actor attribution.
- Embedded metadata: event count, error rate, average latency.

States:
- Loading: spinning spinner with text.
- Error: warning icon, message, retry button.
- Empty: green check icon, message, hint to start simulator.

Stats Bar (top of dashboard):
- Total incident count.
- Critical count (red label).
- Warning count (orange label).
- Auto-refreshes every 3 seconds.


3. API Updates

New functions added to signalforge_mvp/frontend/src/api.ts:

fetchIncidentDetail(incidentId: string): Promise<Incident>
- GET /incidents/{incident_id}
- Returns full incident with timeline.

updateIncidentStatus(incidentId, status, actor, note?): Promise<Incident>
- PATCH /incidents/{incident_id}/status
- Updates status and triggers timeline entry.


4. TypeScript Type Updates

signalforge_mvp/frontend/src/types.ts now exports:

- IncidentTimelineEntry: timestamp, event_type, message, actor, metadata.
- Incident: all fields including timeline[] and updated_at.

Severity and status fields use strict union types for type safety:
- severity: "critical" | "warning" | "info"
- status: "investigating" | "mitigated" | "resolved"


5. Responsive Design

Breakpoints:
- Desktop (> 640px): multi-column grid, side panel overlay, full stats bar.
- Mobile (<= 640px): single-column grid, full-screen panel, compact stats.
- Small mobile (<= 480px): stacked stat boxes, full-width action buttons.

8. Beginner-Friendly Explanation

Imagine SignalForge is a hospital emergency room for your microservices.

The dashboard is the triage board:

- Red cards = critical: the patient needs immediate attention.
- Yellow cards = warning: the patient is stable but needs monitoring.
- Green check = no patients: everyone is healthy.

When you click a card:
- You see the full medical chart: what happened, when, and why.
- You see the timeline: when the incident was created, what evidence was added, when status changed.
- You can update the status: from investigating to mitigated to resolved.

This is how real operations teams track incidents. The dashboard makes it visual and easy to understand.
